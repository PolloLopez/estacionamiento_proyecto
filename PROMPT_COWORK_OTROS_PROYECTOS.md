## Nota sobre documentación

Cada proyecto debería tener estos tres archivos antes de empezar a trabajar con Cowork:
- `CONTEXT.md` — foto fija del sistema (arquitectura, modelos, roles)
- `PENDIENTES.md` — tareas pendientes y resueltas (se actualiza al final de cada sesión)
- `CLAUDE.md` — convenciones de código, git flow, gotchas conocidos

Si el proyecto todavía no los tiene, pedile a Cowork:
"Leé todo el código/docs que hay y armame CONTEXT.md, PENDIENTES.md y CLAUDE.md
siguiendo la convención de documentación de mis proyectos."
